Introduction

息壤 (Breath Flow) 是一项探索人类生理信号与数字生命边界的 VR 生物反馈互动装置。

项目以中国神话中自生自长的"息壤"为灵感，将呼吸——人类最原始的生存信号，转化为虚拟世界的生长动力。用户通过声音传感器捕捉吹气频率，实时驱动 Unity 中的 Shader 动态参数，让花朵呼吸间破土、抽芽、直至盛开。

我们旨在快节奏的焦虑时代，利用生物反馈疗法（Biofeedback），通过视觉上的"生命绽放"诱导用户主动放慢呼吸节奏，在人与自然的数字连接中，重拾内心的秩序与平静。

Ideation

Lofi UI Workflow

核心交互机制：呼吸节奏决定花朵命运

  • 缓慢呼气 → 花朵稳定生长，从花苞到绽放
  • 呼吸过快/过急 → 花瓣被吹散，需要重新开始
  • 完成一朵花 → 收入"你的花园"，可积累不同花朵

这一机制的设计意图：引导用户主动关注呼吸节奏。通过"花瓣飘散"这一自然交互，让用户自然开始缓慢、有节奏的呼吸方式。

Working Prototype

Sensor Selection

本项目理想的传感方案是气流传感器（直接测量呼吸气流量，可获取连续波形）。受限于春节期间硬件采购周期，选择声音传感器作为快速验证方案：

  • 气流传感器（最好）：检测双向气流，连续信号，精度高
  • 胸带式呼吸带：可测呼吸周期，但相较于气流传感器精度更低
  • 声音传感器（当前方案）：做二值判断（阈值>30 为呼气），但最快跑通链路

拾音传感器，使用 Arduino 追踪呼吸声音变化

Reflection & Next Steps

这是一个 1 周的快速技术验证项目，也是我第一次使用 Unity 开发 VR 应用。主要验证了从物理传感器 → Unity Shader → PICO VR 头显的完整技术链路。

如继续迭代：

  1. 替换为气流传感器，获取连续呼吸曲线，实现 Shader 参数的平滑映射
  2. 加入呼吸周期节律分析，系统可反向引导用户调整呼吸节奏，形成 biofeedback 闭环
  3. 实现 Lofi UI 中设计的完整流程（引导、失败状态、花园收集）
  4. 设计对照实验验证"VR 呼吸交互是否真的促进放松"
